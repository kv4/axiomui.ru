Аналитическая платформа нового поколения

Крупная RPA-платформа · 2025–2026 · React + Drupal + PostgreSQL

Суть: Заменили 157 устаревших Drupal-параграфов на 41 React-компонент с JSONB-хранением. Бизнес-пользователи получили drag-and-drop дашборды без участия разработчика.

Контекст и проблема

Крупная платформа автоматизации (RPA) столкнулась с проблемой: бизнес-пользователи нуждались в гибкой визуальной аналитике, но существующее решение было построено на устаревшей архитектуре Drupal Paragraphs. Каждый дашборд был набором параграфов в Drupal-ноде, связанных со сложной системой формул.

  • 157 параграфов в 3 legacy-дашбордах — жёсткая связка данных и представления
  • 75+ формул размазаны по параграфам, нет единого источника истины
  • 75% дашбордов требовали участия разработчика для любого изменения
  • Отсутствие drag-and-drop — любое перемещение виджета = задача в бэклог
  • Невозможность для бизнес-пользователя самостоятельно настроить аналитику

Техническое решение

Архитектура построена на принципе разделения данных и представления. Бэкенд возвращает только сырые числа, клиент форматирует.

Ключевые архитектурные решения

  • Raw Data Principle — бэкенд возвращает только сырые числа, клиент форматирует. data.values никогда не сохраняются в БД.
  • JSONB-хранение — один столбец data JSONB NOT NULL вместо 20+ Drupal-полей. Схема расширяется без миграций.
  • SourceKey иерархия — generic и specific дашборды. Один шаблон на 100+ сущностей.
  • Widget Plugin System — Drupal-плагины с единым контрактом через WidgetPluginInterface.
  • 9 Chart-трансформеров в цепочке ответственности (Chain of Responsibility), конвертирующие любые DSL-данные в Chart.js-формат.
