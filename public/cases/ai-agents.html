ИИ-агенты и автоматизация

Автоматизация разработки через ИИ-агентов

Enterprise-разработка · 2025–2026 · OpenCode + 11 AI-агентов

Суть: 11 ИИ-агентов автоматизировали полный цикл разработки: от планирования до code review. Code review ускорен на 60%, 138 планов задач созданы без единой просрочки.

Контекст и проблема

Команда разработки крупной RPA-платформы столкнулась с вызовами, типичными для enterprise-разработки: высокая сложность кода (PHPStan level 8, строгая типизация), необходимость жёсткой дисциплины (TDD, code review) и распределённая работа в Azure DevOps. Разработчики тратили до 40% времени на рутину.

  • TDD-дисциплина тяжело внедрялась — разработчики «забывали» писать тесты
  • Code review занимал до 30% времени senior-разработчиков
  • Azure DevOps work items требовали ручного обновления статусов
  • Документация отставала от кода

Техническое решение

Развернута платформа OpenCode с системой ИИ-агентов. Вместо того чтобы заставлять разработчиков соблюдать правила вручную, процессы автоматизированы через агентов.

СлойСоставНазначение
MCP-серверы5 шт.Интеграция с внешними системами (Azure DevOps, Playwright, Ant Design и др.)
ИИ-агенты11 шт.Виртуальные роли: Prometheus (планирование), Sisyphus (исполнение), Oracle (аудит), Atlas (качество кода) и др.
Предметные навыки11 шт.dashboard-v2, project-conventions, frontend-conventions, react-testing, run-tests, docker-env, azure-wi-manager и др.

Процесс разработки с агентами

  1. Prometheus (планировщик) — создаёт план задачи
  2. Metis + Momus — gap-анализ и ревью плана
  3. Sisyphus (исполнитель) — пишет код по TDD (RED → GREEN → REFACTOR)
  4. Oracle + Atlas — проверяют compliance и качество кода
  5. azure-wi-manager — автоматически обновляет статус в Azure DevOps
